### STEP 1: PRODUCTION-GRADE INGESTION & FEATURE ENGINEERING
Location Coordinate Target: Sargodha, Pakistan (Latitude: 32.0836, Longitude: 72.6711)

1. Data Source Strategy:
   * Fetch real-time weather and pollutant data simultaneously. 
   * Primary: AQICN API (v2) for air pollutants: PM2.5, PM10, NO2, SO2, CO, O3.
   * Secondary: OpenWeatherMap Air Pollution API and One Call API v3.0 for meteorological metrics: Temperature, Relative Humidity, Wind Speed, Wind Direction, Barometric Pressure, Precipitation.
   * Resiliency: Implement an exponential backoff retry mechanism using the 'tenacity' library to handle API rate-limiting (HTTP 429) and transient network disconnects (HTTP 5xx).

2. Deep Feature Engineering & Math:
   * Temporal Features: Convert UTC timestamps to local Sargodha timezone (PKT). Extract hour, day of week, month, and day of year. Transform cyclical features using sine and cosine encodings to maintain temporal continuity:
     - hour_sin = sin(2 * pi * hour / 24)
     - hour_cos = cos(2 * pi * hour / 24)
     - day_sin = sin(2 * pi * day_of_week / 7)
     - day_cos = cos(2 * pi * day_of_week / 7)
   * Derived Environmental Indicators:
     - AQI Change Rate (First Derivative): Delta AQI / Delta t over a rolling 1-hour, 3-hour, and 6-hour window.
     - Wind-Pollutant Vector Interaction: Decompose Wind Speed and Wind Direction into Eastward (U) and Northward (V) components. Calculate cross-products with key pollutants (PM2.5, PM10) to model horizontal spatial dispersion.
     - Boundary Layer Proxy: Compute temperature-humidity index (THI) to proxy inversion layer trapping effects on stagnant particulate matter.

3. Hopsworks Feature Store Integration:
   * Write `data_pipeline/ingest.py` and `feature_pipeline/register.py`.
   * Programmatically define and initialize a Hopsworks Feature Group:
     - Feature Group Name: `sargodha_aqi_features`
     - Version: 1
     - Primary Keys: `['timestamp']`
     - Event Time: `timestamp`
     - Partition Keys: `['year', 'month']`
   * Schema enforcement: Define precise feature types (Double, Integral, String). Ensure all calculated features pass strict structural validation before write operations.
   * Create a separate backfill pipeline (`data_pipeline/backfill.py`) using historical data from the past 5 years. Implement data-chunking and writing pipelines in batches of 10,000 rows to prevent execution memory overflows.
